1. Architecture d'accès

Tous les appels transitent par un point d'entrée unique — l'API Gateway (port 8080) — qui assure la validation JWT, le rate limiting et le circuit breaker.

Types d'animateurs

La plateforme D2F distingue deux types d'animateurs selon le champ typeFormation :

Animateur interne ESPRIT

  • typeFormation : INTERNE ou EN_LIGNE
  • Champ : animateurs (List<EnseignantRef>, ManyToMany)
  • Données : id, nom, prenom, mail, up, dept
  • Enseignant identifié par son matricule ESPRIT
  • Rôle système : ANIMATEUR

Animateur externe

  • typeFormation : EXTERNE
  • Champs : externeFormateurNom, externeFormateurPrenom, organismeRefExterne
  • Intervenant extérieur à ESPRIT
  • Données stockées en clair dans la formation
  • externeFormateurEmail masqué (RGPD)
⚠️ Les champs externeFormateurNom et externeFormateurPrenom sont accessibles aux rôles ADMIN et D2F uniquement. externeFormateurEmail n'est jamais exposé (RGPD).

4. Règles de gestion

4.1 Validation d'un besoin de formation

Un besoin passe par trois niveaux d'approbation successifs. Un seul refus stoppe le processus.

ÉtapeActeurChampValeurs
1Chef de DépartementapprouveChefDepnull = en attente · true = approuvé · false = rejeté
2CUPapprouveCUPnull = en attente · true = approuvé · false = rejeté
3Administrateur D2FapprouveAdminnull = en attente · true = approuvé · false = rejeté

4.2 Unicité des savoirs

Un Savoir est rattaché soit à une Competence, soit à une SousCompetence — jamais les deux simultanément (contrainte applicative).

4.3 Unicité du niveau par enseignant

Un enseignant ne peut avoir qu'un seul niveau de maîtrise par savoir. Contrainte unique en base : (enseignantId, savoirId).

4.4 Génération automatique des certificats

Un certificat est créé automatiquement pour chaque enseignant dont l'inscription est APPROVED lorsque la formation passe à l'état ACHEVE. Le champ delivered passe à true lors de la remise effective.

7. Pagination, tri et recherche

ParamètreTypeDéfautDescription
pageInteger0Numéro de page (0-based)
sizeInteger20Éléments par page (max 100)
sortStringid,descTri : champ,asc ou champ,desc
Exemple : GET /api/v1/formations?page=0&size=20&sort=dateDebut,desc&etatFormation=PLANIFIE

La recherche textuelle (?keyword=) est insensible à la casse, partielle, portant sur nom, code et description.

Format de réponse paginée

{ "content": [ /* liste d'objets */ ], "totalElements": 154, "totalPages": 8, "number": 0, "size": 20, "first": true, "last": false }
